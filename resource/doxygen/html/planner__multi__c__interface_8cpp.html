Find and return the earliest point in integer time when the request can be satisfied.

Note on semantics: this function returns a time point where the resource state changes. If the number of available resources change at time point t1 and t2 (assuming t2 is greater than t1+2), the possible schedule points that this function can return is either t1 or t2, but not t1+1 nor t1+2 even if these points also satisfy the request.

Parameters
ctxan opaque planner_multi_t context returned from planner_multi_new.
on_or_afteravailable on or after the specified time.
durationrequested duration; must be greater than or equal to 1.
resource_requests64-bit unsigned integer array of size len specifying the requested resource counts.
lenlength of resource_counts and resource_types arrays.
Returns
the earliest time at which the resource request can be satisfied; -1 on error with errno set as follows: EINVAL: invalid argument. ERANGE: resource_counts contain an out-of-range value. ENOENT: no scheduleable point
