int Flux::resource_model::matcher_util_api_t::set_pruning_types_w_spec ( const std::string &  subsystem,
const std::string &  spec 
)

Set prune filters based on spec. The spec should comply with the following format: <HL-resource1:LL-resource1[,HL-resource2:LL-resource2...]...]> Install a planner-based filter at each High-Level (HL) resource vertex which tracks the state of the Low-Level (LL) resources in aggregate, residing under its subtree. If a jobspec requests 1 node with 4 cores, and the visiting compute-node vertex has only a total of 2 available cores in aggregate at its subtree, this filter allows the traverser to prune a further descent to accelerate the search. Use the ALL keyword for HL-resource if you want LL-resource to be tracked at all of the HL-resource vertices. Examples: rack:node,node:core ALL:core,cluster:node,rack:node
